## Architecture decisions

1. **Resend over SendGrid/SES** — free tier sufficient, Vercel-native, clean SDK
2. **Hourly cron (not real-time)** — Yahoo rate-limit constraint; hourly is appropriate for value investing alerts
3. **24h debounce** — prevents spam during sustained price moves
4. **Denormalized email per alert** — no auth system exists; email on each row avoids building a users table
5. **Raw HTML email (not React Email)** — single template doesn't justify 3 extra dependencies

## Deploy checklist for user

1. Sign up at [resend.com](https://resend.com) → get API key
2. Add env vars to Vercel: `RESEND_API_KEY`, `ALERT_FROM_EMAIL`, `CRON_SECRET`
3. Optionally set up DNS (DKIM/SPF) for `alerts@grahamscreener.com`
4. Deploy
5. Verify with `npm run test-email`
